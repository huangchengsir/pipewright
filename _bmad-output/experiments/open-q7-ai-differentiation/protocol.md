# Open Q#7 前置验证实验:AI 失败诊断差异化

> **状态:** 装置就绪(harness + 种子数据集 + 评分),**待真实输入跑真结论**。
> **门槛:** 这是 PRD §9 Open Q#7、产品最大未验证风险,**Epic 7 深水区(7-3 成败 diff / 7-5 反馈闭环)开建前置**。
> **作者:** code-review/实验设计 2026-05-30。

## 0. 一句话

验证「**现成 LLM + 精选知识库 + 成功/失败 diff(FR-22/FR-25)**」是否**显著优于**用户直接把失败日志粘进 ChatGPT;不显著则 AI 差异点必须重定(护城河不在"诊断答案质量",而在别处)。

## 1. 假设(预注册,先写死再跑)

- **H1(差异化成立):** 平台**装配上下文**的诊断(FR-22 结构化 prompt〔技术栈+步骤+自动脱敏日志〕+ FR-25 成功/失败 diff + 知识库检索)在**根因准确率**与**修复可执行性**上**显著优于**把原始失败日志粘进通用对话 LLM(ChatGPT 基线)。
- **H0(零假设):** 无显著差异——通用 LLM 仅凭粘贴日志即可做到同等水平。**若 H0 成立 → AI 差异点重定**(见 §6 决策)。

**关键洞察:** 平台相对"粘进 ChatGPT"的唯一结构性优势,是它能**自动装配人手动粘贴拿不到/拼不出的上下文**——尤其 **FR-25 成功/失败 diff**(上次绿、这次红,之间改了什么)。所以实验的真正命门 = **diff-decisive 案例**:根因不在失败日志里、而在 diff 里的那些(日志只给一个隐晦报错,diff 才暴露"昨天某依赖被 bump")。这类案例是 FR-25 这笔投资值不值的判据。

## 2. 对照臂(递增,隔离价值来自哪一层)

| 臂 | 内容 | 隔离什么 |
|----|------|---------|
| **A — 原始粘贴(ChatGPT 基线)** | prompt = 失败日志原文 + 一句"哪里出错了、怎么修?"。无结构、无额外上下文。**这就是竞品。** | 通用 LLM 裸能力下限 |
| **B — FR-22 平台 prompt** | 复刻 `internal/ai/diagnose.go` `buildDiagnosePrompt`:结构化 + 技术栈/步骤上下文 + 通用错误模式提示 + JSON schema + "假说非结论" + 置信度。**同样的日志内容**,但平台的框架 + 平台自动知道的上下文。 | 仅"更好的 prompt + 栈上下文"的增量 |
| **C — B + FR-25 成功/失败 diff** | 追加"自上次成功运行以来改了什么"块(配置/env/依赖/代码 diff)。**这是平台独有信号**,手动粘贴者拼不出。 | **diff 的边际价值(7-3 是否值得做)** = C−B |
| **D — C + 精选知识库** | 追加 top-k 检索到的过往同类失败+修复(知识库)。 | **KB 的边际价值(7-5 是否值得做)** = D−C |

**最关心的差值:** `C−A` / `D−A`(装配上下文是否打赢裸粘贴)· `C−B`(diff 边际,gate 7-3)· `D−C`(KB 边际,gate 7-5)。

## 3. 指标(每案对 ground-truth 打分)

1. **根因准确率** rootCause ∈ {0,1,2}:2=精确命中真实根因 · 1=相邻/部分 · 0=错或漏。
2. **修复可执行性** fix ∈ {0,1,2}:2=正确且具体可直接执行 · 1=合理但含糊/不全 · 0=错。
3. **证据落地** evidence ∈ {0,1}:是否引用了真正决定性的日志/diff 行。
4. **噪声/幻觉惩罚** penalty ∈ {0,−1}:是否编造不存在的根因/文件/行。

**复合分** = `rootCause*2 + fix*2 + evidence + penalty`(0–11;权重可调,但跑前固定)。

## 4. 评分法(防自欺)

- **盲评 LLM-judge:** 用一个**独立的更强模型**做裁判,逐案对 ground-truth 打分;**剥离臂标签 + 打乱输出顺序**(judge 不知哪段来自哪臂)。
- **人评校准:** 作者对**≥5 案**人工打分,校准 judge 与人评的一致性(Cohen's κ);κ<0.6 则 judge 不可信、回退全人评。
- **同模型控制:** A/B/C/D **必须同 provider/model/temperature**(temp=0),只变上下文框架,不变底座能力——否则测的是模型差异不是装配差异。

## 5. 数据集规范

- **规模:** 作者真实失败日志 **~20 案**(PRD 指定)。本仓 `dataset/seed-cases.json` 先放 **6 个拟真种子**(带 ground-truth)用于**跑通装置 + 演示方法学**,真跑须替换为真实日志。
- **每案 schema:**
  ```json
  {
    "id": "case-01",
    "stack": "node@22 / npm",
    "step": "构建镜像",
    "difficulty": "diff-decisive | log-evident | ambiguous",
    "failureLog": "原始失败日志(多行)",
    "diffSummary": "自上次成功运行的变更摘要(FR-25;arm C/D 用)",
    "kbCases": [{"symptom":"...","rootCause":"...","fix":"..."}],
    "groundTruth": { "rootCause": "...", "decisiveLines": [12], "correctFix": "..." }
  }
  ```
- **难度配比(关键):** 必须含足量 **diff-decisive** 案(根因只在 diff 里、不在日志里)。若数据集全是 log-evident(日志自明),实验会假性得出"无差异"——因为那些案 ChatGPT 裸粘也能答对,平台优势无从体现。**真实采样时刻意纳入"昨天绿今天红、报错隐晦"的案。**
- **控制案:** 留 1–2 个 A 本就该答对的简单案做 sanity(若平台臂在这些上反而更差 = 装置有 bug)。

## 6. "显著优于"阈值与决策规则(预注册)

跑前固定,跑后不准改:

- **显著判据(n≥20):**
  - 平台最佳臂(C 或 D)对 A 的**复合分均值提升 ≥ 25%**,**且**
  - 在"A 未答对"的子集上,平台臂根因准确率胜出 **≥ 60%**(必须赢在 ChatGPT 答错的案上——那才是有意义的案),**且**
  - 配对 Wilcoxon 符号秩检验 **p<0.05**。
- **决策:**
  - **GO**(按计划建 7-3/7-5):`C−A`/`D−A` 过阈 **且** `C−B` 显示 diff 有真实边际(证明 7-3 值得)。
  - **PARTIAL**(重排序):`D−A` 过阈但 `C−B` 可忽略 → KB 有用、diff 没用 → KB 优先,diff 降级/缓做。
  - **NO-GO / 重定差异点**:平台最佳臂 ≈ A → AI 护城河**不在诊断答案质量**;差异化转向**集成与零摩擦**(自动装配上下文免粘贴/脱敏/一键修复在上下文内/反馈飞轮),而非"答得更准"。这本身是有价值的结论——避免在伪护城河上做厚 7-3/7-5。

## 7. 混淆控制

同模型同 temp(=0)· 同日志内容(只变上下文框架)· 盲评 judge · 随机化臂顺序 · sanity 控制案 · judge↔人评 κ 校准。

## 8. ⚠️ 诚实约束(必读)

- **本机跑不出真结论:** 需 ① 作者 ~20 条真实失败日志(种子是拟真,只验装置)② 一个连得通的 LLM。本机 GFW 拦 anthropic/openai、无 Ollama。
- **stub dry-run 只验装置机械正确性**(prompt 装配/4 臂/评分/报告流水线跑通),**不验假设**——stub 返回固定文本,任何"得分"都是占位,**不得当作差异化证据**。
- **真跑建议:** 本地装 Ollama(`qwen2.5-coder` / `llama3.1` 均可)或经 VPN 指一个可达 endpoint;**底座用同一中等模型跑 4 臂**,**judge 用另一个更强模型**。把真实日志填进 `dataset/cases.json`(复制 seed schema),`node harness/run.mjs --provider ollama --model qwen2.5-coder --judge-model ...`。
- **最重要:** 若真跑得出 NO-GO,**这是好消息不是坏消息**——它在投入 7-3/7-5 前就拦下了伪需求,把差异化引向真护城河。诚实记录,不美化。
